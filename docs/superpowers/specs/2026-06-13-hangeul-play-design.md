# 한글 놀이/학습 웹앱 — 설계 문서 (Spec)

- 날짜: 2026-06-13
- 대상 사용자: 48개월(만 4세) 딸아이 (주 사용자) · 부모(콘텐츠 관리·관찰)
- 목적: 종료된 구몬 한글 학습지의 "이야기로 흥미 → 강조된 한글 단어 학습" 경험을, **쉽게 접근 가능한 웹**으로 재현하고, 판매도 가능한 수준의 완성도로 만든다.

## 1. 한눈에 보는 핵심 결정

| 항목 | 결정 |

## 2. 학습 설계

### 2.1 교수 방향
- **통글자 우선 + 자모음 보조**: 4세는 단어를 통째 그림처럼 인식하는 게 흥미·성취가 빠름. 자모(ㄱ,ㅏ…)는 놀이 속에서 자연스럽게 노출.
- **실패 없는 설계(no-fail)**: 틀려도 벌·차감 없음. 부드러운 힌트 → 다시 시도 → 성공 경험으로 마무리.

### 2.2 이야기 중심 레슨 구조 (가장 중요)
구몬에서 따님이 가장 좋아한 "이야기" 경험을 중심에 둔다. **한 번의 "오늘의 모험" = 한 레슨**:

1. **이야기** — 4장면 내외의 짧은 그림 이야기. 음성 내레이션 + 자막. 오늘의 단어 3~4개가 자막에서 강조.
2. **오늘의 단어** — 강조됐던 단어들을 한 번 더 또박또박 만남(그림+소리).
3. **놀이** — 바로 그 단어들로 6종 중 2~3개 놀이.
4. **칭찬·수집** — 별·스티커 획득, "내가 배운 단어"에 추가, 여정 맵 한 칸 전진.

> 핵심: 이야기가 "그날 배울 단어"를 정하고, 놀이가 그것을 복습한다. 단어는 레슨 단위로 묶인다.

## 3. 정보 구조 & 화면

```
홈
├─ 오늘의 모험(레슨)  ← 메인 동선
│   └─ 이야기 → 오늘의 단어 → 놀이 → 칭찬·수집
├─ 자유 놀이          ← 놀이 6종을 직접 골라 가볍게
├─ 스티커 북          ← 모은 스티커 모음
├─ 공주 꾸미기 / 상점 ← 스티커로 옷·악세·배경 구매 + 꾸미기
├─ 여정 맵            ← 레슨 진행을 지도/길로 시각화
└─ 부모 존(잠금)      ← 단어 켜고끄기, 진행 보기, 음성 속도 등 (간단한 게이트)
```

- **홈**: 큰 "오늘의 모험 시작" 버튼 + 공주(현재 꾸민 모습) + 스티커 수.
- **자유 놀이**: 이야기 흐름과 별개로 놀이만. 같은 단어 풀 사용.
- **부모 존**: 어린이가 못 들어가게 간단한 게이트(예: "숫자 3개를 길게 누르기"/나이 입력) — 설정·진행 확인용.

## 4. 이야기 화면 상세

- **장면 일러스트**: A 톤(포근한 3D 그림책)의 전체 화면 장면.
- **음성 + 자막 동시**: TTS가 문장을 읽고, 자막에서 **지금 읽는 단어가 노란 띠로 따라 강조**(가라오케 방식).
- **오늘의 단어 강조**: 목표 단어는 자막에서 더 크게·색(빨강)·살짝 통통 튐. **탭하면** 그림이 뜨고 다시 읽어줌(+작은 반짝임 보상).
- **진행**: 장면 자동 넘김 + 수동(화살표/탭). 다시 듣기 버튼.
- **마무리**: "오늘 만난 단어" 라인업 → "놀이로 익혀볼까?"로 자연스럽게 연결.

## 5. 놀이 6종 + 4세 터치 UX

공통 원칙: **탭 우선**(드래그는 보조, 탭-투-플레이스 대안 제공), **큰 터치 영역**, **2~3 보기**, **즉각 음성·시각 피드백**, **실패 없음**.

| 놀이 | 방식 | 학습 |
|------|------|------|
| 🔊 듣고 그림 찾기 | 소리로 단어 → 여러 그림 중 탭 | 듣기+통글자 |
| 🖼️ 그림 보고 단어 고르기 | 그림 1 → 단어 보기 2~3 중 탭 | 통글자 읽기 |
| 🧩 글자 블록 만들기 | 그림 아래 음절 블록을 탭(또는 드래그)으로 채움 | 자모/음절 |
| 🔎 자음·모음 찾기 | "ㄱ 들어간 그림 모두 탭" 미션 | 낱자 |
| 🃏 카드 짝맞추기 | 그림카드↔글자카드 메모리 | 재미·복습 |
| 📖 이야기 | (위 4장 참조) | 흥미·듣기 |

- **글자 블록**: 4세 손가락엔 드래그가 어려워, 기본은 **빈칸 탭 → 블록 탭** 방식. 드래그도 허용.
- **오답 처리**: 흔들림/부드러운 "다시 해볼까?" 음성 + 정답 살짝 힌트. 결국 성공으로.

## 6. 보상 경제

| 단계 | 트리거 | 보상 |
|------|--------|------|
| 즉각 | 매 정답 | 반짝임·박수 소리·공주/캐릭터 리액션 |
| 스티커 | 모험 1개 완료 | 스티커 1장 → **스티커 북**에 모음 |
| 큰 보상 | 스티커 사용 | **상점**에서 헤어·옷·신발·악세·배경을 하나씩 구매 → **공주 꾸미기** |
| 진행 | 레슨 완료 | **여정 맵**에서 캐릭터가 새 나라로 전진 |

- 4세 친화: **숫자 최소화**, 가격은 스티커 아이콘 1~6개로 표시, 구매는 탭 1번 + 신나는 애니메이션.
- **육성/스탯·실패·차감 없음.** 꾸미는 재미만.
- 확장(나중, 범위 밖): 방 꾸미기, 펫 키우기, 부모용 뱃지.

### 공주 꾸미기 (Model B — 레이어 믹스매치)
- **슬롯별 아이템을 따로 모아 자유 조합**(프린세스메이커 느낌). 스티커로 하나씩 구매.
- 슬롯 9종(초기 ≈83개, 확장): 💇헤어12 · 👗원피스/드레스14 · 👚상의10 · 👖하의10 · 👟신발10 · 👑머리악세8 · 📿목·손악세7 · 🧚등(날개·망토)4 · 🌈배경8.
- 레이어 z-order: 배경 → 등 → 베이스 몸 → 하의 → 상의/원피스 → 신발 → 헤어 → 머리악세 → 목·손악세. (원피스 입으면 상의·하의 숨김)
- 공주는 이야기에도 등장하는 친구. **따님이 이름 지어주기** 가능.

## 7. 데이터 모델

### 7.1 단어 `words.json`
손으로 적는 건 `text`·`theme`·`image` 한 줄. 음절·자모는 코드가 한글 유니코드로 자동 분해.
```json
{
  "id": "apple",
  "text": "사과",
  "theme": "food",
  "image": { "type": "fluent", "name": "red-apple" }
}
```
- 자동 파생: `syllables: ["사","과"]`, `jamo: ["ㅅ","ㅏ","ㄱ","ㅗ","ㅏ"]` (초/중/종성 분해).
- 사진 대체 시: `"image": { "type": "photo", "url": "..." }`.

### 7.2 레슨 `lessons.json`
```json
{
  "id": "fruit-1",
  "title": "과일나라 이야기",
  "theme": "food",
  "story": [ { "sceneImage": "...", "text": "곰돌이가 빨간 사과를 땄어요!", "targets": ["apple"] } ],
  "targetWords": ["apple", "banana", "grape"],
  "games": ["listen-find", "pick-word", "build-word"]
}
```

### 7.3 진행상황 (localStorage)
```json
{
  "stars": 0,
  "stickers": 8,
  "learnedWords": ["apple", "..."],
  "completedLessons": ["fruit-1"],
  "ownedItems": ["hair-01-long-wave", "dress-01-pink-gown", "shoes-01-maryjane", "head-01-tiara"],
  "equipped": {
    "hair": "hair-01-long-wave",
    "dress": "dress-01-pink-gown",
    "top": null, "bottom": null,
    "shoes": "shoes-01-maryjane",
    "head": "head-01-tiara", "neckhand": null, "back": null,
    "background": "bg-01-castle"
  },
  "princessName": "별이"
}
```

## 8. 그림·소리 파이프라인

### 8.1 단어 그림 (3단계 폭포수)
1. **Fluent 3D 일러스트** 매칭(대부분 커버) → 2. 없으면 `words.json`의 **사진 URL**(웹검색으로 채움) → 3. 한 번 정해지면 번들/캐시 → 매번 검색 X, 오프라인 OK.

### 8.2 공주 아트 (AI, 제작 단계 1회성, 레이어 믹스매치)
- **제작 시**: 프롬프트(작성 완료, `docs/assets/princess-image-prompts.md`) → 무료 브라우저 AI 생성 → 배경 투명·오려내기·정렬·파일명 정리 → 번들.
- **실행 시**: 앱은 저장된 PNG 레이어를 z-order로 겹쳐 표시. **AI 호출 0 · 비용 0 · 오프라인 OK.**
- **정렬 핵심**: 고정 포즈 **베이스 1장**을 만들고, 모든 아이템을 *그 베이스 위에 입힌 상태로* 생성 → 그 조각만 투명 오려내기. 같은 베이스에서 나오니 헤어만/옷만/신발만 바꿔도 위치가 맞음. 편집 가능한 도구(ChatGPT/Gemini 이미지 편집) 권장.
- 초기 셋 ≈ **83 아이템**(슬롯 9종) + 베이스 1. 샘플 ≈10장으로 정렬·품질 먼저 확인 후 확장.

### 8.3 소리 (TTS)
- Web Speech API, 한국어 음성 중 자연스러운 것 자동 선택, 아이용 속도 약간 느리게.
- 단어·이야기 읽기. 태블릿 내장 음성이라 오프라인 동작.

## 9. 비주얼 디자인 시스템 ("포근한 크림 + 3D")
- **배경**: 미색~피치 그라데이션(예 `#fff7ec → #ffeede`).
- **폰트**: 본문/숫자 Pretendard, 따뜻한 강조 Gowun Dodum(둥근 톤).
- **컴포넌트**: 둥근 카드, 부드러운 그림자, 살짝 입체적인 버튼(아래쪽 색 보더로 누름맛).
- **아이콘**: 이모지 금지 → **SVG 아이콘**(별·스피커 등) 통일.
- **포인트 컬러**: 따뜻한 오렌지(`#f0a04b`/`#e8852b`), 정답·성장은 그린, 공주는 핑크 계열.

## 10. 기술 스택 & 아키텍처
- **React + Vite**, 정적 SPA.
- **PWA**: 매니페스트 + 서비스워커(에셋 캐시) → 홈화면 설치 + 오프라인.
- **서버 없음**: 단어/레슨 = 번들 JSON, 진행 = localStorage, 이미지 = 번들/캐시, 음성 = 브라우저.
- **부품화**: 놀이 6종 = 컴포넌트, 공통 조각(단어카드·TTS 훅·보상 애니메이션·이미지 리졸버) 재사용.
- 핵심 모듈(독립·테스트 가능):
  - `hangul/` 한글 자모·음절 분해 유틸 (순수 함수, 단위 테스트)
  - `content/` words/lessons 로더·검증
  - `image/` 이미지 리졸버(Fluent→photo)
  - `tts/` 음성 훅(음성 선택·속도·재생)
  - `progress/` localStorage 진행 저장소
  - `reward/` 스티커·상점·꾸미기 상태
  - `games/` 놀이별 컴포넌트
  - `story/` 이야기 플레이어(자막 동기·강조)

---

## 11. 접근성·안전
- 큰 터치 타겟, 단순한 한 화면 한 과제, 소리로 안내(글 못 읽는 아이 배려).
- 실패·시간제한·광고·외부링크·결제 **없음**.
- 상점/부모존은 어린이 오조작 방지 게이트.
- 개인정보 수집 없음(전부 기기 안 localStorage).

---

## 12. 배포
- 무료 정적 호스팅(예: Vercel/Netlify/GitHub Pages) → 깔끔한 URL.
- 태블릿 브라우저에서 "홈 화면에 추가"로 앱처럼 설치(PWA).
- 오프라인: 첫 방문 후 캐시되어 인터넷 없이 동작.

---

## 13. 테스트 전략
- **단위 테스트**: 한글 자모/음절 분해, 단어→놀이 보기 생성, 보상/진행 로직(순수 함수 중심, TDD).
- **콘텐츠 검증**: words/lessons JSON 스키마·이미지 매칭 누락 체크 스크립트.
- **수동 기기 테스트**: 실제 태블릿에서 터치·TTS·오프라인·PWA 설치 확인(체크리스트).

---

## 14. 범위 / 비범위
- **포함(v1)**: 이야기 레슨, 놀이 6종, 8주제 ~80단어, 보상(즉각/스티커/상점/꾸미기/여정맵), **레이어 믹스매치 꾸미기(≈83아이템)**, PWA 배포.
- **비범위(나중)**: 방 꾸미기, 펫, 다기기 동기화/계정, 부모 통계 대시보드, 따라쓰기(쓰기 연습), 옷 색상 무한 변형/세이브 슬롯 여러 개.

## 15. 단계별 구현 순서(제안)
1. **뼈대**: Vite+React+PWA, 디자인 토큰, 홈/네비.
2. **콘텐츠 엔진**: 한글 분해 유틸(+테스트), words/lessons 로더, 이미지 리졸버, TTS 훅.
3. **이야기 플레이어**(핵심): 자막 동기·강조·탭.
4. **놀이 6종**: 듣고찾기·단어고르기부터 → 블록·자모찾기·메모리.
5. **보상**: 즉각 보상 → 스티커/스티커북 → 여정 맵.
6. **공주 꾸미기/상점**: 베이스+샘플 레이어로 정렬 검증 → 슬롯별 레이어 합성·구매·착용 연결.
7. **콘텐츠 채우기**: 80단어 + 첫 레슨 4~6개 + 공주 이미지 생성/정리.
8. **배포·기기 점검**: 호스팅, PWA 설치, 오프라인, 태블릿 테스트.
